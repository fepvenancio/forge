**Step 2: Extend packages/webhook/src/server.ts with push event handler**

Add a `PushPayload` interface:
```typescript
interface PushPayload {
  ref: string;  // "refs/heads/gsd/phase-1-foundation"
  commits: Array<{
    added: string[];
    modified: string[];
    removed: string[];
  }>;
  repository: {
    owner: { login: string };
    name: string;
  };
}
```

Add `extractChangedFiles(payload: PushPayload): string[]` function:
- Flatten all commits' added, modified, removed arrays
- Deduplicate with Set
- Return unique file paths

Add `handlePushEvent(payload: PushPayload): Promise<string>` function:
1. Extract branch name from ref: `payload.ref.replace("refs/heads/", "")`
2. If not GSD branch: return "Skipping non-GSD branch: {ref}"
3. Extract changed files from payload
4. For each changed file:
   - Dynamic import queries: `const queries = await import("../../orchestrator/src/dolt/queries.js")`
   - Call `queries.getFlowsForFile(file)` to find matching flows
   - For each matching flow, call `queries.markFlowStale(flow.id)`
5. Return summary: "Marked {N} flow(s) as stale for {M} changed file(s)"
6. Wrap in try/catch: if Dolt unavailable, log warning and return "Push processed (Dolt unavailable, flow staleness not updated)"

Wire into the webhook route handler:
```typescript
if (event === "push") {
  const result = await handlePushEvent(req.body as PushPayload);
  console.log(result);
  res.status(200).json({ message: result });
  return;
}
```

Place the push handler BEFORE the existing `res.sendStatus(200)` fallback at the end of the route.

Export the new functions: `handlePushEvent`, `extractChangedFiles`
