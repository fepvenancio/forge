Create 3 Python gate scripts following the EXACT pattern from `flow_freshness.py` (same `get_dolt_connection()` helper, same exit code convention, same graceful Dolt degradation).

**packages/gates/touch_map_conflicts.py:**
- Import: `os, sys, subprocess, pymysql`
- Reuse `get_dolt_connection()` pattern from flow_freshness.py (same env vars: DOLT_HOST, DOLT_PORT, DOLT_USER, DOLT_PASSWORD, DOLT_DATABASE)
- `get_pr_branch()`: Get current branch name from `git rev-parse --abbrev-ref HEAD`
- `check_touch_map_conflicts()`:
  1. Get all active phase assignments from Dolt: `SELECT phase_id, branch_name, status FROM phase_assignments WHERE status IN ('assigned','in_progress','pr_open')`
  2. For the current PR branch, get changed files via `git diff --name-only origin/main...HEAD`
  3. For each OTHER active branch, get its changed files via `git diff --name-only origin/main...{branch}` (wrapped in try/except for branches that might not exist locally)
  4. Find file-level overlaps between current branch and other branches
  5. If overlaps found: print conflict table and exit 1
  6. If no overlaps: print "No touch map conflicts" and exit 0
  7. If Dolt unavailable: print "Skipping touch map conflict check (Dolt not available)" and exit 0
- Output format: print `result=pass` or `result=fail` to a file at `$GITHUB_OUTPUT` if that env var exists (for GHA output capture)

**packages/gates/config_validation.py:**
- Import: `os, sys, json`
- NO Dolt needed -- reads `.planning/config.json` directly from the checked-out code
- `validate_config()`:
  1. Read `.planning/config.json` from current directory
  2. Check required fields: `mode` (string), `granularity` (string), `workflow` (object with `research`, `plan_check`, `verifier`, `auto_advance`), `git` (object)
  3. Check `git.branching_strategy === "phase"` specifically
  4. If all valid: print "Config validation passed" and exit 0
  5. If invalid: print each validation error and exit 1
  6. If file not found: print "WARNING: .planning/config.json not found, skipping" and exit 0 (some PRs may not have it)
- Output `result=pass` or `result=fail` to `$GITHUB_OUTPUT`

**packages/gates/cost_audit.py:**
- Import: `os, sys, subprocess, pymysql`
- Reuse `get_dolt_connection()` pattern
- `get_phase_from_branch()`: extract phase number from branch name matching `gsd/phase-{N}-*` pattern, return int or None
- `check_cost_audit()`:
  1. Get current branch phase number
  2. If not a GSD phase branch: print "Not a phase branch, skipping cost audit" and exit 0
  3. Query Dolt: `SELECT SUM(cost_usd) as total_cost FROM developer_costs WHERE phase_id = ?`
  4. If no data: print "WARNING: No cost data for phase {N}. Cost tracking not yet active." and exit 0 (Phase 4 builds cost tracking)
  5. If data exists: print cost summary and exit 0 (no budget threshold enforcement yet -- that's Phase 4 COST-04)
  6. If Dolt unavailable: print "Skipping cost audit (Dolt not available)" and exit 0
- Output `result=pass`, `result=warn`, or `result=skip` to `$GITHUB_OUTPUT`

Each script must have `if __name__ == "__main__": main()` entry point and use `sys.exit(0)` for pass, `sys.exit(1)` for fail.

Create `.github/workflows/forge-gates.yml` -- a single workflow with 6 jobs (5 parallel gate checks + 1 post-results summary).

```yaml
name: Forge Gates
on:
  pull_request:
    branches: [main]

jobs:
  flow-freshness:
    runs-on: ubuntu-latest
    outputs:
      result: ${{ steps.check.outputs.result }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install -r packages/gates/requirements.txt
      - id: check
        run: |
          if python packages/gates/flow_freshness.py; then
            echo "result=pass" >> "$GITHUB_OUTPUT"
          else
            echo "result=fail" >> "$GITHUB_OUTPUT"
          fi
        env:
          DOLT_HOST: ${{ secrets.DOLT_HOST }}
          DOLT_PORT: ${{ secrets.DOLT_PORT }}
          DOLT_DATABASE: forge

  conflict-check:
    runs-on: ubuntu-latest
    outputs:
      result: ${{ steps.check.outputs.result }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install -r packages/gates/requirements.txt
      - id: check
        run: |
          if python packages/gates/touch_map_conflicts.py; then
            echo "result=pass" >> "$GITHUB_OUTPUT"
          else
            echo "result=fail" >> "$GITHUB_OUTPUT"
          fi
        env:
          DOLT_HOST: ${{ secrets.DOLT_HOST }}
          DOLT_PORT: ${{ secrets.DOLT_PORT }}
          DOLT_DATABASE: forge

  config-validation:
    runs-on: ubuntu-latest
    outputs:
      result: ${{ steps.check.outputs.result }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - id: check
        run: |
          if python packages/gates/config_validation.py; then
            echo "result=pass" >> "$GITHUB_OUTPUT"
          else
            echo "result=fail" >> "$GITHUB_OUTPUT"
          fi

  test-suite:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - run: pnpm install --frozen-lockfile
      - run: pnpm build
      - run: pnpm test

  cost-audit:
    runs-on: ubuntu-latest
    outputs:
      result: ${{ steps.check.outputs.result }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install -r packages/gates/requirements.txt
      - id: check
        run: |
          if python packages/gates/cost_audit.py; then
            echo "result=pass" >> "$GITHUB_OUTPUT"
          else
            echo "result=fail" >> "$GITHUB_OUTPUT"
          fi
        env:
          DOLT_HOST: ${{ secrets.DOLT_HOST }}
          DOLT_PORT: ${{ secrets.DOLT_PORT }}
          DOLT_DATABASE: forge

  post-results:
    needs: [flow-freshness, conflict-check, config-validation, test-suite, cost-audit]
    if: always()
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
    steps:
      - uses: actions/github-script@v7
        with:
          script: |
            const marker = '<!-- forge-gates-report -->';
            const results = {
              'Flow Freshness': '${{ needs.flow-freshness.outputs.result }}' || 'skipped',
              'Conflict Check': '${{ needs.conflict-check.outputs.result }}' || 'skipped',
              'Config Validation': '${{ needs.config-validation.outputs.result }}' || 'skipped',
              'Test Suite': '${{ needs.test-suite.result }}',
              'Cost Audit': '${{ needs.cost-audit.outputs.result }}' || 'skipped',
            };
            const icon = v => v === 'pass' || v === 'success' ? ':white_check_mark:' : v === 'skipped' ? ':fast_forward:' : ':x:';
            const rows = Object.entries(results).map(([k,v]) => `| ${icon(v)} ${k} | ${v} |`).join('\n');
            const body = [
              marker,
              '## Forge CI Gates',
              '',
              '| Gate | Result |',
              '|------|--------|',
              rows,
              '',
              `*Updated: ${new Date().toISOString()}*`,
            ].join('\n');
            const { data: comments } = await github.rest.issues.listComments({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
            });
            const existing = comments.find(c => c.body && c.body.includes(marker));
            if (existing) {
              await github.rest.issues.updateComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                comment_id: existing.id,
                body,
              });
            } else {
              await github.rest.issues.createComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                issue_number: context.issue.number,
                body,
              });
            }
```

Key points:
- Use `actions/github-script@v7` for the post-results step (not raw octokit -- handles auth automatically)
- The `<!-- forge-gates-report -->` marker enables update-or-create pattern (Pitfall 2 from research)
- `if: always()` on post-results ensures comment is posted even when a gate fails
- `permissions: pull-requests: write` is required for the comment posting step
- test-suite job does NOT output a custom result -- uses `needs.test-suite.result` (built-in GHA job conclusion: success/failure)
- All Dolt-dependent gates pass `DOLT_HOST`, `DOLT_PORT`, `DOLT_DATABASE` from secrets
- config-validation does NOT need Dolt (reads local file)
